# CodeGraphX Usage with Coding Agents

## 🚀 Purpose

**CodeGraphX** gives coding agents (like Gemini CLI, Claude Code, Copilot CLI, Cursor) an instant, up-to-date graph of your entire project—without expensive file scanning or wasted tokens.  
Just run CodeGraphX after cloning the repo: your agent will instantly understand your structure, relationships, and changes.  
No database, no web server, no cloud dependency—everything is local, fast, and agent-optimized.

---

## ⚡️ Quick Start (After Cloning)

1. **Install dependencies (if not present):**
   ```sh
   npm install
   ```

2. **Generate your initial code graph:**
   ```sh
   npx codegraphx init
   # or if installed globally:
   codegraphx init
   ```

   Optionally, start live mode so your graph stays instantly up-to-date:
   ```sh
   npx codegraphx watch
   ```

3. **That’s It!**
   - Your entire codebase is now represented as efficient graph files in `.codegraphx/`.
   - No agent ever needs to manually scan hundreds of files again.

---

## 🧠 How Your Coding Agent Should Use CodeGraphX

When the agent session starts, read these files instead of scanning the raw codebase:

| Purpose                           | CodeGraphX File                   | Description                                                      |
|------------------------------------|-----------------------------------|------------------------------------------------------------------|
| **Project file/entity overview**   | `.codegraphx/file_index.toon`     | Single-line summary per file: type, count, overview              |
| **What changed recently**          | `.codegraphx/CHANGELOG.toon`      | Per-session/commit log of code changes                           |
| **Instant symbol lookup**          | `.codegraphx/symbols.json`        | Symbol → location mapping (function, class, etc)                 |
| **Deep dependencies/edges**        | `.codegraphx/codegraph.toon`      | Full inter-entity graph (calls, imports, etc), highly compressed |
| **Bloom filter for fast checks**   | `.codegraphx/symbols.bloom`       | Fast “does symbol exist?” before deeper lookups                  |
| **Rich visualization**             | `.codegraphx/codegraph.html`      | Graphical D3-based browser dashboard (open in browser)           |

> ⚡ The agent should always start by reading `file_index.toon` and `CHANGELOG.toon` for fast orientation.
> 
> Only request `codegraph.toon` for deep dependency questions or impact queries.

## 🟢 Typical Workflow

- **Initial project scan:**  
   `npx codegraphx init`
  
- **(Optional) Live updates on save:**  
   `npx codegraphx watch`
  
- **Updating after refactoring:**  
   `npx codegraphx scan`  
   or just rerun `init`.

- **After commit (if hook installed):**
   - `.codegraphx/CHANGELOG.toon` is auto-updated with a summary of what changed.

**No manual scanning** is needed for your coding agent. Just read the files above and you have a full, up-to-date map.

## 🎯 Example GEMINI.md (Agent Instructions)

```
# Project Context (Auto-generated by CodeGraphX)

Before scanning the codebase, use these files for instant orientation:

1. Read `.codegraphx/file_index.toon` (one line per file, type/summary)
2. Read `.codegraphx/CHANGELOG.toon` (per-session/commit change log)
3. Use `.codegraphx/symbols.json` to look up any function/class
4. Use `.codegraphx/codegraph.toon` for deep dependency/calls analysis

Checking whether a symbol exists? Use Bloom filter in `.codegraphx/symbols.bloom`.  
Visual exploration: Open `.codegraphx/codegraph.html` in your browser.

Never rescan the raw codebase unless absolutely necessary!
```

## 🏗️ FAQ

**Q:** What happens if I add or remove a file?  
**A:** Run `npx codegraphx scan` or keep `codegraphx watch` running. The graph files stay perfectly synced.

**Q:** Which agents are supported?  
**A:** Any agent or tool that can read files! Gemini CLI, Claude Code, Copilot CLI, Cursor, and more.

**Q:** What is TOON format and why?  
**A:** It’s an open, ultra-token-efficient format for LLM agents. Smaller file, same info, faster agent context loading.

**Q:** Is there a config?  
**A:** Yes, customize `.codegraphxrc` or `codegraphx.config.json` to include/exclude files, languages, output, etc.

## 🛡️ Security/Privacy

- All data stays local: CodeGraphX never sends anything to the cloud.
- You can `.gitignore` or commit the `.codegraphx/` directory (recommended: commit CHANGELOG, others optional).

---

## 🧩 Advanced/Dev Usage

- See full CLI help: `npx codegraphx --help`
- Live dashboard: `npx codegraphx dashboard`
- Agent/impact queries: `npx codegraphx query name`, `npx codegraphx impact foo`
